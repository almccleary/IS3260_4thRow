Web Site Deliverable #2

Submit URL via D2L Dropbox for the Group. Only one member of the group needs to submit the URL and your zipped website files.

Design Specifications:
  • Follow HTML5 specifications on every page.
  • Each page must validate using  the W3C Markup Validation Service:
  • Follow html rules and syntax. Documents must be well-formed
  • You may use tables or CSS for the layout of your site.  Your design must be your own, your original work.  You may not use a template from the Internet, or content management, or website frameworks.
  • !DOCTYPE identifier must appear before the root element
  • Lowercase tags and attribute names
  • Attribute values must always be quoted
  • All attributes must have a value
  • Every tag must be closed or terminated
  • Implement an intuitive file folder structure. Use only lower case characters in file names. Do not use spaces when naming files or folders.
  • Use relative paths where appropriate.
  • Include alt and title attributes for all images.
  • Implement an external cascading style sheet and uses styles where appropriate..
  • Use only Web-safe colors and specify all colors in hexadecimal.
  • Comment your code.

Required Elements:

  • Implement templates. This may be Dreamweaver templates which I recommend for our projects or an HTML template. Regardless if you use CSS layout or Tables you must use a template.
  • Page titles on every page. (Define title tags)
  • Implement at minimum four graphics on the site including a Logo that uniquely identifies the site..
  • Implement an external stylesheet. Include a minimum of 5 styles however all formatting should be in the stylesheet.
  • Include a credits page that gives credit for graphics, text, etc. Be sure to add a disclaimer that this is a student project for a college course. The page should include a short description of the team members.
  • Implement a Sitemap page.
  • Implement a feedback form that can be used to provide you with feedback regarding the site design and content. Use formbuddy to implement the processing of the form.
  • Implement an intuitive menu.
  • A minimum of 8 customer content pages pages for the site is required
  • A minimum total of 11 pages for the site.  This includes the three required pages credits, sitemap and feedback.
  • Post site to the studentweb.kennesaw.edu server. No other server may be used.
  • Content text will be the current information already published on the current site.  No Lorem ipsum.
  • Submit the URL . Zip all files and folders needed to run the site and submit with url.
